# Frontend Design System and Templates

## Purpose

The WordPress front end must support technical B2B evaluation, not only marketing presentation. Templates should help buyers quickly understand product function, standards relevance, configuration needs, evidence level, and inquiry path.

## Design Tokens

- Spacing: 4, 8, 12, 16, 24, 32, 48, 64 px scale.
- Typography: restrained technical style, clear hierarchy, no oversized hero type inside dense product tools.
- Color: professional neutral base with limited accent colors for CTA, status, warnings, and evidence labels.
- Cards: use for repeated product/resource items only; avoid card-inside-card layouts.
- Tables: responsive, sortable where useful, and readable on mobile.
- Breakpoints: mobile, tablet, desktop, wide desktop.
- Image ratios: product main image, gallery image, diagram, application image, social preview, PDF cover.
- Accessibility: visible focus states, keyboard navigation, text contrast, form labels, alt text.
