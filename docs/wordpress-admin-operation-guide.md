# WordPress 后台操作指南

## 目的

本指南用于后续在独立 WordPress 测试站中安全搭建 KingPo 中文医疗测试设备网站框架。

网站定位：脑机接口与有源植入式医疗器械测试设备研发方向展示站 + 技术能力介绍站 + 定制开发询盘站。

本指南只适用于测试站 / staging site，不适用于任何正式网站后台。

## 绝对边界

不得操作以下正式站后台：

- dgkingpo.com
- kingpo.hk
- batterytestingmachine.com
- sinuotek.com
- 任何未明确声明为测试站的 WordPress 后台

不得把 WordPress 后台地址、用户名、密码、cookie、token 或 nonce 写入 GitHub 仓库。

## 禁止操作

- 不修改正式网站。
- 不改 DNS。
- 不绑定正式域名。
- 不创建 301。
- 不迁移旧站。
- 不删除页面、文章、媒体文件、主题、插件或用户，除非用户明确批准。
- 不修改管理员主账号。
- 不发布未经确认的正式内容。
- 不把研发方向写成成熟现货产品。
- 不公开老板未确认的研发进展。
- 不写未经确认的产品参数、标准符合性、认证、客户案例、库存、价格或交期。
- 不安装来源不明、破解、盗版或高风险插件/主题。
- 不提交 sitemap，不接入 Search Console，不主动请求搜索引擎收录。

## 允许操作范围

在用户明确提供的测试站后台中，可以进行：

1. 登录测试站 WordPress 后台。
2. 检查 WordPress 版本、主题、插件、语言、固定链接、首页设置。
3. 创建测试站基础页面。
4. 创建导航菜单。
5. 创建研发方向占位页。
6. 创建医疗测试设备分类页。
7. 创建标准与测试页面。
8. 创建技术文章和资料下载占位页面。
9. 设置 noindex，阻止搜索引擎收录测试站。
10. 配置基础 SEO 插件，但不得提交正式索引。
11. 创建测试询盘表单，但不得使用正式客户数据。
12. 上传可公开测试图片或占位图。
13. 使用 Gutenberg / 区块编辑器搭建页面框架。

## 进入后台后的第一步检查流程

进入后台后，必须先检查并输出：

1. WordPress 版本
2. 当前主题
3. 当前已安装插件
4. 当前语言
5. 当前固定链接结构
6. 当前首页设置
7. 当前菜单结构
8. 是否已设置 noindex
9. 是否有缓存插件
10. 是否有 SEO 插件
11. 是否有表单插件
12. 是否有页面构建器
13. 是否适合继续搭建测试站框架

如果发现这是正式站、旧站、已有大量正式内容站或不确定站点身份，必须停止并询问用户。

## 必须等待用户确认的操作

以下操作必须先说明目的并等待用户确认：

- 更改全站设置
- 更改首页设置
- 更改固定链接结构
- 安装、启用、停用、删除或替换插件
- 安装、启用、删除或替换主题
- 修改主题文件或插件文件
- 大幅调整菜单结构
- 发布页面或文章
- 上传内部图片、PDF、系统图或技术资料
- 创建真实邮件发送表单
- 修改 SMTP 设置
- 添加缓存、安全、SEO 或图片优化插件
- 删除任何内容
- 对外提交索引、sitemap、Search Console 或统计代码

## 默认页面状态

所有页面默认创建为 Draft / 草稿。

测试站页面可以用于预览，但必须 noindex。

不得将占位内容发布为正式内容。

## Noindex 设置要求

测试站必须阻止搜索引擎收录。

优先设置：

1. WordPress 后台 > 设置 > 阅读 > 建议搜索引擎不索引本站点。
2. 如安装 SEO 插件，再检查全站 noindex 或测试站 noindex 设置。
3. 如需要 robots.txt 或服务器级限制，必须先请用户确认。

完成后需验证页面是否包含 noindex 或站点是否启用 discouraging search engines。

## 插件策略

插件必须来自 WordPress 官方插件库、插件官网或用户明确批准的可信来源。

安装前必须列出插件清单和用途，等待确认。

可考虑但需确认的插件：

| 插件 | 用途 | 备注 |
| --- | --- | --- |
| Advanced Custom Fields | 结构化字段：研发状态、来源状态、标准映射 | 推荐，但需确认安装 |
| Rank Math 或 Yoast SEO | SEO metadata、noindex 控制 | 二选一，需确认 |
| Fluent Forms 或 Contact Form 7 | 测试询盘表单 | 二选一，需确认 |
| WP Mail SMTP | 邮件发送测试 | 只有需要测试邮件时安装 |
| 缓存插件 | 性能优化 | 框架稳定后再考虑 |
| 安全插件 | 测试站安全 | 避免锁死后台访问 |
| 图片优化插件 | 图片压缩 | 等图片体系明确后再考虑 |
| Elementor | 首页或联系页可选构建器 | 不作为核心数据层 |

不得安装来源不明、破解、盗版或高风险插件/主题。

## 推荐主题策略

当前阶段不购买重型商业主题。

优先使用轻量主题，并配合 Gutenberg / ACF / 自定义模板框架。

主题更换必须先确认。

## 第一批研发方向占位页

默认创建为草稿：

- 脑机接口测试设备
- 有源植入式医疗器械测试设备
- EEG 动态模拟闭环测试仪
- 电生理动态模拟闭环测试仪
- 电极耐压自动测试仪
- 神经刺激器电极组织接口网络模块

如标题与基础页面重复，应先判断页面用途，避免重复页面混乱。

## 研发方向占位页必须包含

每个研发方向占位页必须显示：

- 研发方向说明
- 参数待确认提示
- 标准映射待确认提示
- 技术方案咨询 CTA
- 不得写成现货产品
- 不得写“已完全符合某标准”
- 不得写“已交付客户”
- 不得写“已通过认证”

推荐占位说明：

当前页面为测试站占位内容。本方向用于研发方向说明和定制开发评估，具体产品状态、技术参数、标准版本、条款方向、样机状态和公开范围需要人工确认。

## 后台执行顺序建议

1. 确认站点身份为测试站。
2. 完成后台检查清单。
3. 确认 noindex 状态。
4. 向用户报告主题、插件、固定链接、首页、菜单现状。
5. 请求用户确认是否可以调整全站设置、安装必要插件或更换主题。
6. 创建基础草稿页面。
7. 创建研发方向草稿占位页。
8. 创建或规划菜单结构。
9. 设置首页和菜单前再次确认。
10. 输出预览地址、后台地址、页面清单、菜单清单、待确认事项。

## 阶段输出格式

每次后台操作阶段完成后，输出：

1. 测试站预览地址
2. WordPress 后台地址，不含账号密码
3. 已安装主题和插件
4. 已创建页面清单
5. 已创建导航结构
6. 已创建分类结构
7. 已创建模板/组件清单
8. 当前哪些页面只是占位
9. 哪些内容需要用户确认
10. 下一步建议先完善哪 3 个页面

## 安全结论

WordPress 后台操作以“测试站框架搭建”为目标，不以正式上线为目标。任何可能影响生产、索引、正式域名、用户账号、真实客户数据或公开研发进展的操作，都必须先暂停并等待用户确认。
