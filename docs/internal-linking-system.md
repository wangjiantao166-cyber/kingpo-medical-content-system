# Internal Linking System

## Purpose

The site should behave like a connected product knowledge graph, not a set of isolated product pages.

Strong internal linking should help users move between:

- Products
- Product categories
- Applications
- Standards
- Technical articles
- PDFs
- Inquiry forms

## Required Links by Page Type

### Product Category Page

Required outbound links:

- 3-12 products or R&D directions in the category
- 2-6 related applications
- 2-8 related standards
- 2-6 technical articles
- Main inquiry page
- PDF download page when available

### Product Page

Required outbound links:

- Parent category
- Related standards
- Related applications
- Related products
- Relevant technical articles
- PDF download card when approved
- Inquiry form

### Standard Page

Required outbound links:

- Related product categories
- Related products
- Related applications
- Technical articles explaining test context
- Inquiry page

### Application Page

Required outbound links:

- Recommended products
- Related standards
- Related articles
- Product category
- Inquiry page

### Technical Article

Required outbound links:

- At least one product or category
- At least one application or standard when relevant
- Inquiry page
- Related articles

## Anchor Text Rules

Use descriptive Chinese anchor text.

Good examples:

- 脑机接口闭环测试设备
- ISO 14708 相关测试设备方向
- 有源植入式医疗器械测试方案咨询

Avoid vague anchor text:

- 点击这里
- 查看更多
- 了解详情

## Link Risk Rules

Do not link an unconfirmed R&D reference page as if it is an existing product.

If the linked page is a draft, internal-only, or needs confirmation, do not expose it on public pages.

Do not create standard links that imply full compliance unless there is verified evidence.

## Orphan Page Checks

Every public page must have:

- At least one inbound link from a category, standard, application, article, or navigation page
- At least three meaningful outbound internal links unless it is a legal/contact page
