# 产品资料与来源要求

## 目的

本文件定义 KingPo 脑机接口与有源植入式医疗器械测试设备中文站的资料来源归属、标准映射、核对规则和人工确认要求。

中文站内容必须真实、可追溯、适合中文 B2B 医疗测试设备采购、研发验证、注册检验和第三方检测用户。不得编造产品参数、标准版本、条款号、测试条件、客户案例、认证资质、注册结论、临床结论、价格、库存或交期。

## 项目边界

本项目是全新的独立中文站：

- 不迁移旧站。
- 不替换旧站。
- 不创建 301 跳转。
- 不修改任何现有网站。
- 不登录任何现有网站后台。
- 不绕过 robots.txt 或访问限制。
- 不进行高频请求或一次性全站爬取。
- 现有网站只作为只读资料来源或只读参考来源。

## 第一阶段资料优先级

### P0：新上传表格 / PDF 清单 / 用户确认资料

第一阶段核心产品路线以用户新上传的表格、PDF 清单、产品路线资料和人工确认信息为最高优先级。

### P1：标准映射资料

每个产品方向必须先做标准映射，再决定是否可以写页面。

标准映射必须包含：

- Standard Number
- Standard Name
- Organization
- Relevance to Product Direction
- Test Object
- Possible Test Item
- Page Use: 主标准 / 辅助标准 / 背景标准 / 不适用
- Evidence Source
- Human Verification Required

国际标准体系详见：`docs/international-standards-landscape.md`。

如果标准适用性不确定，标记为“需要人工确认”，不要写进最终产品页。

以下网站可以作为 KingPo 产品事实来源或旧资料参考来源：

## 禁止编造的信息

不得编造或推断以下内容：

- 产品技术参数
- 标准版本
- 标准条款
- 测试条件
- 适用标准或符合性声明
- 认证资质
- 注册结论
- 临床结论
- 客户名称
- 客户案例
- 测试能力
- 测试精度
- 价格
- 库存
- 交期
- 未确认的现货状态

## 数据状态标签

建议对每个关键字段使用以下状态：

- `已确认`：有明确 P0 / A 级来源或用户确认。
- `标准映射待确认`：标准相关性、版本、条款或测试条件仍需确认。
- `P0 来源待核对`：来自新上传表格/PDF，但字段仍需复核。
- `自有来源待核对`：来自自有或优先资料源，但字段仍需复核。
- `旧资料参考`：来自旧资料来源，只能作为线索，需要复核。
- `同行参考`：仅来自同行网站或公开竞品资料，只能用于结构、用户需求、市场学习或研发参考。
- `研发参考 / 需要人工确认`：竞品或路线资料有相关方向，但 KingPo 是否已有产品、是否可售、参数是否成立需人工确认。
- `存在冲突`：多个来源信息不一致。
- `需要人工确认`：无法从资料中确认。

当前阶段只建立医疗专站资料来源规则、国际标准映射体系和测试站部署计划，不录入正式网站，不发布页面，不创建 301，不连接服务器，不创建网站代码，不生成最终产品页面。
