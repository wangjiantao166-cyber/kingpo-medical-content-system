# information-architecture-navigation

## Purpose

Own product domains, cross taxonomies, URL hierarchy, navigation, breadcrumbs, page creation thresholds, and content clusters.

## Use Conditions

Use for site structure, navigation, category planning, application/standard entry points, and page intent mapping.

## Do Not Use Conditions

Do not finalize canonical/noindex decisions or publish navigation to production.

## Accepted Input Schema

- site map
- product taxonomy inventory
- page intent registry

## Output Schema

Navigation plan, URL plan, taxonomy decisions, and page creation/merge decisions.

## Blocking Conditions

Block if categories mix unrelated dimensions or a proposed page lacks buyer intent and unique value.

## Handoff Rules

Send URL/indexation details to `technical-seo-link-graph` and implementation needs to `wordpress-content-model-admin`.
