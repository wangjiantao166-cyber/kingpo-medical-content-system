# Navigation and Mega Menu Architect

## Use When

Use this skill when designing the website header, product navigation, mega menu, mobile menu, breadcrumbs, footer links, or category navigation.

## Goal

Turn the SEO architecture into user-friendly navigation that helps buyers and search engines understand the site structure.

## Required Navigation Areas

- Header navigation
- Product mega menu
- Applications menu
- Standards menu
- Technical resources menu
- Breadcrumbs
- Footer navigation
- Mobile navigation

## Recommended Header Navigation

- Products
- Applications
- Standards
- Technical Resources
- About KingPo
- Contact

## Mega Menu Rules

The Products menu should expose major L1 categories and selected L2 subcategories, but should not list every product model.

Applications and Standards should be visible as independent discovery paths, not hidden inside blog categories.

## Breadcrumb Rules

Product page: Home > Products > L1 Category > L2 Category > Product

Standard page: Home > Standards > Standard Family > Standard Page

Application page: Home > Applications > Application Category > Application Page

## Coordination

This agent depends on:

- `seo-site-architecture-planner` for approved hierarchy
- `content-cluster-planner` for cluster priorities
- `internal-linking-architect` for link behavior
- `google-seo-eeat-content` for user-first navigation labels

## Prohibited

- Do not overload the header with too many links.
- Do not hide important L1 pages.
- Do not use vague labels that buyers cannot understand.
- Do not use misleading category names for SEO only.
