# Inquiry Conversion

## Use When

Use this skill when designing product CTAs, inquiry forms, contact pages, PDF request forms, or custom-development consultation flows.

## Inquiry Types

- Quotation request
- Technical consultation
- Custom development evaluation
- Standard/test method consultation
- PDF/catalog request
- Sample fixture discussion

## Required Technical Fields

- Product or direction
- Test object
- Reference standard
- Test item
- Sample status
- Parameter range
- Fixture/customization need
- Required document
- Timeline if user provides it

## CTA Wording Examples

- 提交测试需求
- 咨询定制测试方案
- 获取产品资料
- 确认适用标准与测试条件
- 提交样品与参数需求

Avoid e-commerce wording such as `立即购买` unless there is a confirmed sales model.

## Output

- CTA recommendation
- Form field plan
- Product-specific inquiry questions
- Spam reduction notes
- Routing notes

## Prohibited

- Do not promise delivery time, stock, certification, or test result.
- Do not collect unnecessary sensitive information.
- Do not make forms so generic that technical staff cannot qualify the inquiry.
