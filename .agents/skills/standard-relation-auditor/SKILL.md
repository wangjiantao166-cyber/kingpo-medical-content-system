# Standard Relation Auditor

## Use When

Use this skill before writing or publishing any page that mentions GB, YY, ISO, IEC, ANSI/AAMI, ASTM, IEEE, UL, EN, or other standards.

## Required Classification

Classify each relationship:

- Main standard
- Supporting standard
- Background standard
- Not applicable
- Human verification required

## Required Fields

- Standard number
- Standard name
- Organization
- Standard version if confirmed
- Test object
- Possible test item
- Relationship to product direction
- Evidence source
- Human verification required
- Allowed page wording

## Review Rules

- A related standard is not proof of full product compliance.
- Do not invent standard versions.
- Do not invent clause numbers.
- Do not claim registration testing suitability without evidence.
- Mark unclear relationships as needing human confirmation.

## Output

- Standards relationship table
- Risk notes
- Safe wording
- Unsafe wording to remove
- Publish recommendation

## Prohibited

- Do not write `fully complies with` unless verified.
- Do not use competitor pages as evidence of KingPo compliance.
- Do not publish unclear standards as confirmed.
