# Content Cluster Planner

## Use When

Use this skill when planning a product category, SEO topic cluster, application hub, standard hub, or publishing roadmap.

## Goal

Create content clusters that support rankings and buyer navigation by connecting high-value commercial pages with supporting informational pages.

## Cluster Components

Each cluster should include:

- L1 pillar page
- L2 subcategory pages
- Product pages
- Application pages
- Standard pages
- Technical articles
- PDF/download resources
- Inquiry CTA

## Example: Battery Safety Testing Cluster

- L1: Battery Testing Equipment
- L2: Battery Safety Test Equipment
- L2: Battery Crush Test Equipment
- L2: Battery Nail Penetration Test Equipment
- L3: Product model pages
- Article: How to choose battery crush test equipment
- Standard/application: Lithium battery abuse testing
- CTA: Submit battery test requirements

## Example: Medical Electrical Safety Cluster

- L1: Medical Test Equipment
- L2: Medical Electrical Safety Test Equipment
- L2: Defibrillation Compatibility Test Equipment
- L2: Active Implantable Medical Device Test Equipment
- Standard pages: IEC 60601, ISO 14708, related standards
- Application pages: R&D validation, third-party lab testing
- CTA: Submit medical test equipment requirements

## Output

- Cluster name
- Pillar page
- Supporting pages
- Required source checks
- Required internal links
- Suggested anchor text
- Priority order
- Missing content gaps

## Coordination

Work with `seo-site-architecture-planner`, `product-knowledge-base`, `application-page-system`, `standard-relation-auditor`, `internal-linking-architect`, and `google-seo-eeat-content`.

## Prohibited

- Do not create clusters with no product support.
- Do not create articles that never link to commercial pages.
- Do not create standard pages that imply unsupported compliance.
