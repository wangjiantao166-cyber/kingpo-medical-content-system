# Google SEO and E-E-A-T Content

## Use When

Use this skill when creating, rewriting, reviewing, or refreshing product pages, product categories, application pages, standard pages, technical articles, FAQs, SEO metadata, and AI-generated product drafts.

## Core Rule

Write for real customers first. Use SEO to help search engines understand the page, not to inflate the page with keywords.

## E-E-A-T Application

### Experience

Show practical testing and buyer context:

- Test object
- Sample type
- Test purpose
- Application scenario
- Selection/configuration questions
- Inquiry preparation checklist

### Expertise

Show technical accuracy:

- Explain test principle when useful.
- Include parameters only with sources.
- Classify standards carefully.
- Mark uncertain data as needing confirmation.

### Authoritativeness

Use KingPo-owned, user-confirmed, or approved sources. Competitor references are for market learning only and must not become KingPo facts.

### Trustworthiness

Avoid unsupported claims. Clearly handle product status, source status, public-use asset status, and human verification status.

## S-CTs: Search Content Trust Signals

Check whether the page includes relevant trust signals:

- Product status
- Source ownership
- Human verification status
- Standard relationship type
- Test object and test item clarity
- Evidence-backed parameters
- Approved assets
- PDF version/date
- Buyer-focused FAQ
- Internal links
- Technical inquiry CTA
- Last review date

## Google SEO Checks

Review:

- Unique H1
- Unique SEO title
- Unique meta description
- Descriptive URL slug
- Logical heading structure
- Helpful internal links
- Image alt text
- Mobile readability
- Avoidance of duplicate/thin content
- Structured data only when visible content supports it

## Content Style

Content should be professional, clear, trustworthy, specific to the product or test scenario, useful for overseas B2B buyers, and cautious with standards and compliance.

Avoid keyword stuffing, empty marketing claims, overly academic writing, repeated template sections, and unsupported certification or compliance claims.

## Output

Return:

- SEO/E-E-A-T review
- Missing S-CTs
- Content risks
- Suggested improvements
- Safer standard wording
- Metadata recommendations
- Publish readiness recommendation
