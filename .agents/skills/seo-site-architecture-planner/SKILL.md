# SEO Site Architecture Planner

## Use When

Use this skill before building the site, redesigning navigation, adding large product groups, or creating a new SEO content cluster.

## Goal

Create a professional site architecture map that prevents orphan pages and supports product, application, standard, and technical-resource discovery.

## Required Inputs

- Product categories
- Existing source websites
- Target markets
- Priority products
- Standards and applications
- Competitor structure observations for reference only

## Output

Produce a site architecture table with:

- Level: L0, L1, L2, L3
- Page type
- Page title
- Search intent
- Primary keyword
- Secondary keywords
- Suggested URL slug
- Parent page
- Child pages
- Required inbound links
- Required outbound links
- Index/noindex recommendation
- Required source support

## KingPo Recommended Top-Level Hubs

- Battery Testing Equipment
- Environmental Test Chambers
- Safety Compliance Test Equipment
- IEC Test Equipment
- Medical Test Equipment
- IP Waterproof / Dustproof Test Equipment
- Plug, Socket and Gauge Test Equipment
- Flame and Fire Resistance Test Equipment
- Applications
- Standards
- Technical Resources

## Coordination

After creating the architecture map:

1. Send product/category assumptions to `product-knowledge-base`.
2. Send standard-related pages to `standard-relation-auditor`.
3. Send hub and cluster structure to `content-cluster-planner`.
4. Send navigation needs to `navigation-mega-menu-architect`.
5. Send link rules to `internal-linking-architect`.

## Prohibited

- Do not create pages only because keywords exist.
- Do not create architecture that lacks source support.
- Do not copy another website's structure exactly.
- Do not create deep pages without a clear parent and inbound link path.
