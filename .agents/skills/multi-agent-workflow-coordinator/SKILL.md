# Multi-Agent Workflow Coordinator

## Use When

Use this skill at the beginning of any complex task involving multiple areas, such as site architecture, new product clusters, WordPress setup, product publishing, homepage planning, or SEO audits.

## Goal

Ensure all agents work in the correct order, share outputs, and do not create conflicting decisions.

## Coordination Principles

- Product facts come before content writing.
- Product status comes before publishing decisions.
- Standards review comes before compliance wording.
- Architecture comes before navigation.
- Cluster planning comes before article generation.
- Asset approval comes before image/PDF publication.
- SEO/E-E-A-T review comes before final QA.
- WordPress draft creation comes before publishing.

## Standard Workflow for Site Architecture

1. `seo-site-architecture-planner`
2. `product-knowledge-base`
3. `content-cluster-planner`
4. `navigation-mega-menu-architect`
5. `homepage-authority-hub`
6. `standard-relation-auditor`
7. `google-seo-eeat-content`
8. `internal-linking-architect`
9. `orphan-link-auditor`
10. `product-publish-qa`

## Standard Workflow for Product Publishing

1. `product-knowledge-base`
2. `product-lifecycle-manager`
3. `standard-relation-auditor`
4. `asset-library-manager`
5. `ai-product-page-generator`
6. `google-seo-eeat-content`
7. `content-cluster-planner`
8. `internal-linking-architect`
9. `wordpress-draft-publisher`
10. `product-publish-qa`

## Required Handoff Format

Each agent should pass:

- Decisions made
- Evidence/source references
- Open questions
- Risk flags
- Required next agent
- Publish readiness status

## Conflict Resolution

If agents disagree:

- Source evidence beats generated content.
- User-confirmed facts beat inferred facts.
- Standards auditor overrides compliance wording.
- Product lifecycle status overrides CTA and indexability.
- QA can block publication.

## Prohibited

- Do not publish based on one agent's output alone.
- Do not skip source verification for speed.
- Do not let SEO suggestions override factual accuracy.
- Do not let design decisions hide important technical content.
