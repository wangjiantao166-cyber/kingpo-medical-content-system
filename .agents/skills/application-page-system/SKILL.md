# Application Page System

## Use When

Use this skill when creating pages around testing scenarios, solutions, applications, or customer use cases.

Examples:

- BCI closed-loop testing
- Active implantable medical device verification testing
- Neural stimulation signal simulation
- Transcutaneous energy transfer testing
- Defibrillation compatibility testing
- TMS compatibility testing

## Page Goal

An application page should help a technical buyer understand:

- What is being tested
- Why the test matters
- Which standards may be relevant
- Which KingPo products or directions may help
- What information they should provide in an inquiry

## Required Sections

1. H1
2. Application summary
3. Target users
4. Test object and sample type
5. Common test requirements
6. Related standards
7. Related products or R&D directions
8. Testing workflow or system diagram note
9. Inquiry preparation checklist
10. FAQ
11. CTA

## Internal Links

Link to:

- Product categories
- Product pages
- Standard pages
- Technical articles
- Inquiry page

## Wording Rules

Use cautious technical language. Do not imply certified performance, regulatory approval, or clinical outcome.

## Output

- Application page brief
- SEO title
- Meta description
- URL slug
- Related product links
- Related standard links
- FAQ
- Inquiry CTA

## Prohibited

- Do not fabricate customer cases.
- Do not copy competitor application text.
- Do not write unsupported test results.
