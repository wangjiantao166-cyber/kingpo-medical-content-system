# Application Page System

## Use When

Use this skill when creating pages around testing scenarios, solutions, applications, or customer use cases.

Examples include BCI closed-loop testing, active implantable medical device verification testing, neural stimulation signal simulation, transcutaneous energy transfer testing, defibrillation compatibility testing, and TMS compatibility testing.

## Page Goal

An application page should help a technical buyer understand what is being tested, why the test matters, which standards may be relevant, which KingPo products or directions may help, and what information they should provide in an inquiry.

## Required Sections

1. H1
2. Application summary
3. Target users
4. Test object and sample type
5. Common test requirements
6. Related standards
7. Related products or R&D directions
8. Testing workflow or system diagram note
9. Inquiry preparation checklist
10. FAQ
11. CTA

## Internal Links

Link to product categories, product pages, standard pages, technical articles, and inquiry page.

## Output

- Application page brief
- SEO title
- Meta description
- URL slug
- Related product links
- Related standard links
- FAQ
- Inquiry CTA

## Prohibited

- Do not fabricate customer cases.
- Do not copy competitor application text.
- Do not write unsupported test results.
