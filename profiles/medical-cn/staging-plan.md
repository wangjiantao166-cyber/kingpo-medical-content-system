# 测试站与预发布规划

## 核心原则

本项目必须先在测试站或预发布环境中完成规划、搭建、内容录入、模板验证、SEO 检查和询盘流程测试。

当前阶段不连接服务器，不修改任何正式网站，不更改 DNS，不发布页面，不创建 301 跳转，不复制同行内容。

## 新站隔离要求

中文站应作为全新的独立医疗测试设备中文站建设。

- 不迁移旧站。
- 不替换旧站。
- 不删除旧站页面。
- 不更改旧站 URL。
- 不修改旧站后台、主题、插件、数据库、表单或邮件设置。
- 不创建旧站到新站的 301 跳转。
- 现有网站只作为只读资料来源或只读同行参考来源。

## 第二阶段测试站部署方式

推荐部署方式：

1. 独立子域名测试站，例如 `staging.example.com`。
2. 独立 WordPress 安装，使用单独数据库、单独文件目录、单独后台账号和单独上传目录。
3. 如服务器尚未确认，可先使用临时预览环境验证医疗专站信息架构、字段结构和样板页内容。

测试站不得覆盖任何现有生产站文件或数据库。

## WordPress 插件清单

当前阶段只规划插件，不安装插件。测试站可优先考虑：

- 自定义字段插件，用于产品参数、标准条款方向、来源记录、人工确认状态、研发参考状态和 PDF 字段。
- SEO 插件，用于 SEO Title、Meta Description、Schema、面包屑和 XML Sitemap 规划。
- 表单插件，用于产品询盘、产品名称自动带入、测试项目字段和测试收件邮箱验证。
- SMTP 测试插件，用于测试站邮件发送验证。
- 安全插件，用于登录保护、基础加固和临时账号检查。
- 备份插件，用于测试站文件和数据库备份。
- 图片优化插件，用于 WebP、压缩和 alt text 工作流。
- 缓存/性能插件，用于测试站速度验证。

404 监控可以作为 QA 思路保留，但当前阶段不把 Redirection、301 或重定向类插件列为必要插件。本项目是新独立中文站，不迁移旧站，不替换旧站，不创建 301。

7. 植入式医疗器械除颤兼容测试仪
8. 大功率电场模拟器
9. 超声兼容测试仪
10. 有源植入式器械机械应力综合测试套件
11. 压力循环测试系统
12. 电极耐压自动测试仪
13. 静磁场测试系统
14. 时变磁场测试仪
15. 心律管理器械低频电磁抗扰度测试仪

## 自有资料源采集范围

第一阶段优先采集新上传表格 / PDF 清单 / 用户确认资料。

自有或优先资料源：

- `dgkingpo.com`
- `kingpo.hk`
- `batterytestingmachine.com`，历史来源，不作为第一阶段重点
- `syrianetf.com`，仅作为旧资料参考源，不能作为生产目标

第一轮只采集与医疗测试设备、脑机接口、有源植入式器械、医疗电气安全、神经刺激、植入式器械兼容性测试相关的页面或文件线索。

## 同行参考源学习范围

同行参考源只能学习医疗测试设备分类、医疗电气安全和有源植入式器械测试场景、页面结构、FAQ 方向、用户需求、研发参考、关键词方向、应用场景表达、下载资料模块形式和询盘 CTA 形式。

不得复制同行正文、图片、PDF、参数表、认证声明、客户案例、品牌表达、型号体系、软件界面、图纸、截图或页面布局。不得把同行数据作为 KingPo 产品事实。

## Fluke Biomedical 和 WhaleTeq 医疗竞品学习范围

Fluke Biomedical / 福禄克和 WhaleTeq 仅用于医疗电气安全与医疗检测设备方向的公开竞品学习、产品路线参考、研发方向分析和内容策略参考。

允许学习：

- 医疗电气安全测试产品分类
- IEC 60601 相关测试场景
- 患者漏电流 / 应用部分相关测试场景
- ECG / 心电相关测试方向
- 高频电刀 / 电外科设备测试方向
- 除颤 / AED / 生理信号相关测试方向
- 页面结构、FAQ 方向、用户需求、测试应用表达

如 Fluke 或 WhaleTeq 有 KingPo 暂未确认销售的产品或功能，必须标记为“研发参考 / 需要人工确认”，不得发布为 KingPo 现货产品页面。

## 必须人工确认的内容

以下内容进入样板页前必须人工确认或标记来源：

- 每个样板页对应产品是否为 KingPo 已有产品、研发中产品或研发参考方向
- 产品型号和中文名称
- 技术参数
- 标准号、标准版本和条款方向
- GB 16174 / YY 0989 / YY/T 1996 / ISO 14708 的适用范围
- EEG / EMG / EOG / LFP / Spike / ECoG 信号范围和通道数
- 闭环刺激、经皮能量传递、除颤兼容、TMS、超声、EMC 等测试能力
- 电极、电导线、绝缘、耐压、组织接口测试参数
- 认证资质、注册结论、临床结论和客户案例是否允许公开
- 产品图片、PDF、系统图是否可用于新中文站
- 询盘邮箱、表单字段、公司名称和品牌标准写法

## 阶段流程

### 阶段 1：规划

输出医疗专站架构、产品分类、内容规范、数据要求、插件规划和协作流程。

### 阶段 2：测试站准备

在获得批准后准备独立测试环境。确认数据库、文件目录、后台账号和访问权限均与现有网站隔离。

### 阶段 3：结构与模板验证

在测试站中验证医疗测试设备产品分类、产品详情页模板、询盘 CTA、自定义字段、SEO 字段、图片与 PDF 附件展示、移动端和桌面端布局。

### 阶段 4：内容录入与来源核对

所有产品数据必须来自可核对来源。缺失、冲突或不确定的信息标记为“需要人工确认”。研发方向标记为“研发参考 / 需要人工确认”。

### 阶段 5：QA 检查

上线前至少检查内容准确性、来源记录、SEO 标题与描述、内部链接、表单提交、邮件送达、移动端布局、桌面端布局、图片 alt、PDF 下载、重复内容风险、安全风险、医疗合规表达风险和竞品复制风险。

### 阶段 6：人工审批

正式发布、DNS 变更、服务器连接、生产环境插件安装、页面发布或任何影响现有网站的动作，都必须等待人工明确批准。

## 本阶段禁止操作清单

当前阶段禁止：

- 连接正式服务器
- 修改任何现有网站
- 登录旧站后台
- 修改 DNS
- 发布页面
- 创建 301 跳转
- 安装或配置 Redirection / 301 相关插件作为当前必要项
- 批量导入产品到生产站
- 修改旧站 WordPress 设置
- 修改旧站数据库
- 修改旧站询盘表单或邮箱
- 复制同行内容、图片、PDF、参数表、型号体系、图纸、截图、软件界面或布局
- 把同行或竞品参数写成 KingPo 参数
- 生成未经来源确认的产品技术参数
- 把研发参考产品写成 KingPo 已有现货产品

## 需要人工提供的信息

进入测试站实施前，需要确认：

- 新上传表格/PDF 清单是否为第一阶段核心路线
- 第一批样板页是否按本文件优先级执行
- 测试站域名或临时访问地址
- 是否允许创建独立 WordPress 安装
- 测试站服务器或主机方案
- 测试站管理员账号交付方式
- 表单测试收件邮箱
- 是否需要密码保护测试站
- 是否有现成品牌素材、产品图片和 PDF
