# Agent 协作规划

## 协作目标

本文件定义 KingPo 全新独立中文站项目的 agent 协作方式。所有 agent 的工作都必须服从项目边界：新站独立建设，旧站只读参考，不连接服务器，不修改正式网站，不改 DNS，不发布页面，不创建 301 跳转，除非未来获得人工明确批准。

## 总体边界

- 当前项目是全新的独立中文站。
- 现有网站只作为只读资料来源。
- 不迁移旧站。
- 不替换旧站。
- 不创建 301 跳转。
- 不修改任何旧站页面、数据库、主题、插件、表单、邮箱或 URL。
- 所有关键产品数据必须有来源或标记为“需要人工确认”。

## Agent 角色

### Master Agent

负责总体协调、范围控制、阶段拆分和最终汇总。

职责：

- 确认当前任务是否在允许范围内
- 分配 SEO、资料、内容、WordPress、QA、安全等工作
- 汇总结果和风险
- 在任何重大动作前要求人工确认

### Product Data Agent

负责资料收集和数据核对。

职责：

- 从只读来源整理产品事实
- 标记来源 URL、PDF、图片或用户文件
- 比对冲突信息
- 将不确定字段标记为“需要人工确认”
- 不编造参数、标准、认证、客户、价格、库存或交期

### SEO & Architecture Agent

负责中文站结构和 SEO 规划。

职责：

- 规划导航、分类、应用页、标准页和技术资料页
- 设计页面 SEO 字段
- 提出内部链接建议
- 避免薄内容和关键词堆砌
- 不创建或建议当前阶段应用 301 跳转

### Content Rewrite Agent

负责将来源资料改写为中文 B2B 页面内容。

职责：

- 基于真实来源重写中文内容
- 面向采购、选型、技术评估场景表达
- 保留事实边界
- 不直接复制旧站内容
- 不补写未确认能力或案例

### WordPress Developer Agent

负责 WordPress 产品目录、字段和模板规划。

职责：

- 优先规划独立 WordPress 测试站
- 规划产品 CPT、分类法、ACF 或自定义字段
- 规划产品模板和询盘 CTA
- 不连接生产服务器
- 不安装或修改生产插件
- 不修改生产主题或数据库

### Baidu Technical SEO Agent

负责中文搜索与技术 SEO。

职责：

- 规划 URL、标题、描述、图片 alt、Schema 和 sitemap
- 检查索引风险、重复内容风险和内部链接
- 区分测试站和正式站索引策略
- 不执行 DNS 变更或 301 配置

### QA Agent

负责上线前检查和页面质量验证。

职责：

- 检查内容准确性
- 检查来源记录
- 检查移动端和桌面端布局
- 检查表单、PDF、图片 alt、内部链接
- 检查 404、重复内容和 SEO 风险
- 未获批准不得发布页面

### Security Agent

负责安全、备份、权限和上线风险。

职责：

- 检查是否存在凭据泄露风险
- 建议临时账号和最小权限
- 要求重大操作前备份与回滚方案
- 禁止在 GitHub 中保存密码、密钥、数据库文件或插件授权码
- 涉及生产环境时必须先要求人工批准

## 推荐工作流

1. Master Agent 确认任务范围。
2. Product Data Agent 收集和标记来源。
3. SEO & Architecture Agent 规划页面结构和 SEO 字段。
4. Content Rewrite Agent 输出中文页面草稿。
5. WordPress Developer Agent 映射字段和模板需求。
6. Baidu Technical SEO Agent 检查 URL、索引和内部链接。
7. QA Agent 检查内容、布局、表单和链接。
8. Security Agent 检查权限、备份和风险。
9. Master Agent 汇总并等待人工批准。

## 阶段交付物

### 规划阶段

- 站点架构
- 测试站计划
- WordPress 插件与字段计划
- 数据来源要求
- Agent 协作计划

### 内容阶段

- 产品资料表
- 产品页草稿
- SEO 元数据
- FAQ
- 图片 alt
- 内部链接建议

### 测试站阶段

- WordPress 结构验证
- 产品模板验证
- 样例产品录入
- 表单测试
- 移动端和桌面端检查

### 上线前阶段

- QA 报告
- 安全检查
- 备份确认
- 回滚计划
- 人工最终批准

## 升级与停止条件

任何 agent 遇到以下情况必须停止并交给 Master Agent 汇总，等待人工确认：

- 需要连接服务器
- 需要登录任何现有网站后台
- 需要修改 DNS
- 需要发布页面
- 需要创建 301 跳转
- 需要修改生产数据库
- 需要导入产品到生产站
- 产品关键数据无法确认
- 发现凭据、客户隐私或敏感信息

## 输出格式建议

每次阶段性输出应包含：

- 已完成事项
- 使用的数据来源
- 需要人工确认的信息
- 风险与限制
- 下一步建议
- 是否涉及生产环境：是 / 否
- 是否需要人工批准：是 / 否
