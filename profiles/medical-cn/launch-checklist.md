# 中文站上线检查清单

## 一、上线前原则

本项目所有正式上线操作必须经过人工确认。

上线前不得直接修改正式网站、正式 DNS、正式数据库或正式服务器配置。

所有结构性修改应先在测试站完成，包括：

* 主题修改
* 插件安装
* 产品模板修改
* ACF 字段修改
* 询盘表单配置
* URL 结构修改
* 301 跳转配置
* 批量产品导入
* 缓存和性能优化

## 二、内容检查

每个重点页面上线前必须检查：

* H1 是否清晰
* 一句话产品定位是否准确
* 核心信息是否放在产品概述前
* 产品概述是否专业
* 适用标准是否准确
* 测试原理是否合理
* 测试项目是否真实
* 技术参数是否有来源
* 应用场景是否符合实际客户
* FAQ 是否回答真实问题
* 询盘 CTA 是否清楚
* 是否存在错别字
* 是否存在机械翻译
* 是否存在空泛营销话术

## 三、技术信息检查

以下内容必须人工确认或有明确来源：

* 产品型号
* 适用标准
* 标准版本
* 测试范围
* 测试项目
* 技术参数
* 电源要求
* 尺寸和重量
* 软件功能
* 配置选项
* 附件清单
* 校准证书说明
* 客户案例
* 认证资质

如资料不明确，必须标记为“需要人工确认”。

## 四、SEO 检查

每个重点页面上线前必须检查：

* SEO Title
* Meta Description
* URL Slug
* 核心关键词
* 相关关键词
* 图片文件名
* 图片 Alt Text
* 内部链接
* FAQ
* Schema 建议
* Canonical 设置
* 是否加入 sitemap
* 是否存在重复内容风险

## 五、页面体验检查

上线前需要检查：

* PC 端布局
* 手机端布局
* 平板端布局
* 导航菜单
* 面包屑
* 产品分类
* 产品详情页
* 图片显示
* 表格显示
* PDF 下载按钮
* 询盘 CTA
* 页面加载速度
* 字体显示
* 按钮点击
* 页面是否错位

## 六、询盘表单检查

必须测试：

* 表单是否能提交
* 邮件是否能收到
* 邮件是否进入垃圾箱
* 收件人邮箱是否正确
* 客户提交信息是否完整
* 上传附件是否正常
* 手机端表单是否正常
* 表单成功提示是否清楚
* 防垃圾设置是否影响客户提交

## 七、链接检查

上线前必须检查：

* 首页链接
* 产品分类链接
* 产品详情页链接
* 标准文章链接
* 案例文章链接
* PDF 下载链接
* 图片链接
* 内链
* 页脚链接
* 联系方式链接
* 旧 URL 是否有跳转
* 是否存在 404

## 八、301 和旧 URL 检查

如涉及旧站迁移，必须准备：

* 旧 URL 列表
* 新 URL 列表
* 301 跳转表
* 保留页面清单
* 合并页面清单
* 删除页面清单
* 404 风险清单

不得在没有跳转计划的情况下删除旧页面。

## 九、技术 SEO 检查

上线前检查：

* sitemap.xml
* robots.txt
* canonical
* noindex 设置
* 面包屑
* Schema
* Open Graph
* 页面状态码
* 页面加载速度
* 图片压缩
* WebP
* 缓存设置
* 移动端可访问性

## 十、安全和备份检查

上线前必须完成：

* 旧站完整备份
* 数据库备份
* 文件备份
* 服务器快照
* 插件更新检查
* 主题更新检查
* 管理员账号检查
* 临时账号权限检查
* SSH Key 检查
* 密码安全检查
* 回滚方案确认

## 十一、上线确认

正式上线前必须确认：

* 测试站已完成检查
* 关键页面已人工确认
* 询盘表单已测试
* 邮件收发已测试
* SEO 字段已完成
* 旧 URL 跳转已准备
* 备份已完成
* 回滚方案已准备
* DNS 修改方案已确认
* 最终发布已获得人工批准

## 十二、上线后检查

正式上线后 24 小时内检查：

* 首页是否正常打开
* 重点产品页是否正常打开
* 手机端是否正常
* 询盘表单是否正常
* 邮件是否正常收到
* Sitemap 是否可访问
* Robots 是否正确
* 是否出现大量 404
* 是否出现服务器错误
* 页面速度是否异常
* 百度站长或搜索工具是否能抓取

## 十三、禁止操作

未经明确批准，不得执行：

* 切换正式 DNS
* 删除正式站页面
* 批量发布页面
* 批量修改 URL
* 批量导入产品
* 修改正式数据库
* 删除插件
* 更换主题
* 覆盖正式站文件
* 修改正式站询盘邮箱
