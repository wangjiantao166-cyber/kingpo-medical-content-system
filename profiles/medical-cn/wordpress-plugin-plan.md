# WordPress 插件与医疗测试设备产品目录规划

## 技术路线

中文站技术路线优先采用 WordPress。当前文件仅规划 WordPress 产品目录与插件能力，不创建网站代码，不连接服务器，不安装插件，不修改任何正式网站。

## 项目边界

本项目是全新的独立中文医疗测试设备站：

- 不迁移旧站。
- 不替换旧站。
- 不创建 301 跳转。
- 不修改旧站后台、主题、插件、数据库或文件。
- 现有网站只作为只读资料来源或只读同行参考来源。

## 插件规划目标

WordPress 层需要支持 BCI 与有源植入式医疗器械测试设备网站的核心内容管理：

- 医疗测试设备产品目录
- 产品分类
- 产品状态：已有产品 / 研发中 / 研发参考 / 需要人工确认
- 适用标准与条款方向
- 测试对象与样品类型
- 测试项目
- 技术参数
- 信号 / 电气 / 机械 / 兼容性条件
- PDF 资料
- 图片与 alt text
- 询盘 CTA
- SEO 元数据
- 相关产品
- FAQ
- 来源记录与人工确认状态

## 当前阶段建议插件类型

当前阶段只规划测试站插件，不安装插件，不配置生产站插件。

建议优先考虑：

- 自定义字段插件，用于产品参数、标准条款方向、来源记录、人工确认状态、研发参考状态和 PDF 字段。
- SEO 插件，用于 SEO Title、Meta Description、Schema、面包屑和 XML Sitemap 规划。
- 表单插件，用于产品询盘、产品名称自动带入、测试项目字段和测试收件邮箱验证。
- SMTP 测试插件，用于测试站邮件发送验证。
- 安全插件，用于登录保护、基础加固和临时账号检查。
- 备份插件，用于测试站文件和数据库备份。
- 图片优化插件，用于 WebP、压缩和 alt text 工作流。
- 缓存/性能插件，用于测试站速度验证。

404 监控可以作为 QA 思路保留，但当前阶段不把 Redirection、301 或重定向类插件列为必要插件。本项目是新独立中文站，不迁移旧站，不替换旧站，不创建 301。

用于管理每台设备、模块、测试系统或产品系列。

建议字段：

- 产品中文名称
- 产品英文名称
- 产品型号
- 产品状态
- 一句话产品定位
- 核心信息
- 产品概述
- 适用标准与条款方向
- 测试原理
- 测试对象与样品类型
- 测试项目
- 技术参数
- 研发验证 / 注册检验 / 第三方检测应用
- 选型与配置建议
- 相关产品
- FAQ
- 询盘 CTA 文案
- 产品图片
- PDF 下载
- SEO Title
- Meta Description
- URL Slug
- 核心关键词
- 相关关键词
- 图片 Alt Text
- Schema 建议
- 数据来源
- 人工确认状态

### 产品分类 Product Category

第一阶段建议分类：

- 脑机接口测试设备
- 有源植入式医疗器械测试设备
- 神经刺激器测试设备
- 植入式起搏器 / ICD 测试设备
- 电生理信号模拟与闭环测试
- 经皮能量传递测试设备
- 除颤兼容测试设备
- 磁场 / TMS / 超声 / EMC 兼容测试设备
- 电极、电导线与组织接口测试设备
- 医疗电气安全测试设备

### 标准 Standard

用于按标准组织页面和产品关联。

建议字段：

- 标准编号
- 标准名称
- 标准版本
- 条款方向
- 适用测试对象
- 相关测试项目
- 关联产品
- 来源
- 人工确认状态

### 测试场景 Application / Test Scenario

用于医疗研发和检测场景入口。

建议字段：

- 场景名称
- 目标用户
- 常见测试需求
- 适用产品
- 相关标准
- 研发验证 / 注册检验 / 第三方检测说明
- 询盘 CTA

## 插件能力建议

可通过自定义插件、ACF 或两者结合实现。正式选择前需结合测试站环境确认。

建议能力：

- 注册产品自定义文章类型
- 注册产品分类法
- 注册标准和测试场景分类法
- 管理产品字段
- 管理来源字段
- 管理人工确认和研发参考状态
- 输出结构化产品模板所需数据
- 支持产品与标准、测试场景、相关产品关联
- 支持 SEO 插件字段映射
- 支持图片 alt 和 PDF 附件字段

## 禁止自动生成的信息

不得在插件、模板或内容中自动编造：

- 产品技术参数
- 适用标准或标准版本
- 标准条款
- 认证资质
- 注册结论
- 临床结论
- 客户案例
- 测试能力
- 价格
- 库存
- 交期
- 保修承诺
- 未确认的型号关系
- 未确认的现货状态

## 开发顺序建议

1. 确认第一批 20 个医疗样板页和字段结构。
2. 确认是否使用 ACF 或自定义插件字段。
3. 在测试站实现产品 CPT、标准和测试场景字段。
4. 创建医疗产品详情页模板。
5. 录入少量样例产品进行验证。
6. 检查 SEO、移动端、询盘 CTA、来源记录和医疗合规表达风险。
7. 通过 QA 后再考虑批量内容流程。
